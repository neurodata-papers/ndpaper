Reproduciblity Guidelines

All of the scientific discovery and infrastructure presented in the associated paper are executed in accordance with open and reproducible science guidelines. Therefore, all of our code and figures are able to be generated automatically to allow scientists to verify and expand on these results. This website contains content for reproduciblity of the content in the paper.

Infrastructure

Without a terascale infrastructure, neuroscientists have to build custom hardware and software to load sub-cubes of data into RAM (for example, in MATLAB or Python), because the amount of RAM on a single laptop, workstation or compute note is typically orders of magnitude smaller than the overall size of the dataset. This is a departure from a typical big data paradigm where the processing demand is for many images or files, each of which easily fits into RAM (e.g., a 2D image).

Images

You can reproduce our Read/Write throughput and the tilecache benchmarks using AWS EC2 instances. The software stack can be installed on the EC2 instance using an install scripts. After installing the back-end, you can run the benchmarks in the paper using the benchmark scripts provided. The benchmark scripts are configurable to run the benchmark tests for different test configurations, for example running the benckmark for 32 threads. The results of these benchmarks are stored in CSV files and you can use provided R scripts to generate the figures as shown in the paper.
